To use this component you will also need to use the <SidebarBlock />
component located inside the <Sidebar /> component folder.
These sidebar blocks are passed in as children to the <Sidebar />
component and they then render the individual blocks of the sidebar.

### Code:

```javascript
import Sidebar from '@department-of-veterans-affairs/formation-react/Sidebar';
import SidebarBlock from '@department-of-veterans-affairs/formation-react/SidebarBlock';

const firstSidebarBlock = {
  heading: 'This is the heading',
  content: <div>This is some content</div>,
  headingLevel: 3,
};

<Sidebar>
  <SidebarBlock
    heading={firstSidebarBlock.heading}
    content={firstSidebarBlock.content}
    headingLevel={firstSidebarBlock.headingLevel}
  />
</Sidebar>;
```

### Rendered Component:

<div class="vads-l-col--12 medium-screen:vads-l-col--4">
  <div class="medium-screen:vads-u-padding-left--4">
    <div class="vads-u-margin-bottom--5">
      <h3 class="vads-u-margin-top--0 vads-u-padding-bottom--1p5 vads-u-border-bottom--3px vads-u-border-color--primary">
        This is the heading
      </h3>
      <div>This is some content</div>
    </div>
  </div>
</div>

### Sidebar Block Props

| prop         |      type      | required | notes                                                                                      |
| ------------ | :------------: | -------: | ------------------------------------------------------------------------------------------ |
| heading      |     string     |      yes | a string that renders as the text of the heading in the sidebar block                      |
| content      | React Fragment |      yes | a React fragment that renders as the content of the sidebar block                          |
| headingLevel |      int       |       no | an alternative heading level if you want the header rendered as something other than an H2 |
